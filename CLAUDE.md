# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 全局协作规则

### 沟通风格
- 默认用中文沟通，除非任务明确要求英文
- 先讲结论，再讲关键原因
- 复杂任务先给简短计划，再开始执行
- 不确定时说明假设和风险，不要装作确定

### 工作方式
- 动手前先阅读相关文件和已有约定
- 修改范围要尽量小，不做与任务无关的重构
- 遇到用户已经修改过的文件，先理解现状再继续
- 发现需求含糊时，先提出最关键的问题
- 能直接验证的结果，优先用命令或测试验证

### 安全边界
- 不要删除用户数据、历史记录、配置文件和密钥
- 不要自动执行 `git push`、部署、发布和破坏性迁移
- 需要大范围改动时，先说明影响范围
- 命令失败时不要硬猜，先读错误信息

### 交付标准
- 说明改了什么
- 说明验证了什么
- 测试不能跑时说明原因
- 标出仍然存在的风险和下一步建议

### 长期记忆维护
- 反复出现的偏好，沉淀到本文件
- 只在单个项目生效的规则，写到项目 CLAUDE.md
- 具体业务规范和长流程，放到项目 `docs` 目录
